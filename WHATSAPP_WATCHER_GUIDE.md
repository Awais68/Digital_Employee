## 📋 Overview

WhatsApp Watcher monitors your WhatsApp messages via Twilio API and automatically creates task files in the `/Needs_Action` folder.

### Features

- ✅ **Twilio API Integration** - Connects to WhatsApp Business API
- ✅ **Auto Message Fetching** - Checks every 30 seconds (configurable)
- ✅ **Task File Creation** - Converts messages to structured `.md` files
- ✅ **Deduplication** - Tracks processed messages to avoid duplicates
- ✅ **Priority Detection** - Auto-detects urgent vs normal messages
- ✅ **tmux Support** - Runs continuously in background
- ✅ **Comprehensive Logging** - Full audit trail in `/Logs/`
- ✅ **Enabled/Disabled Flag** - Only active when `WHATSAPP_ENABLED=true`

## 🚀 Quick Start

### 1. Configure Twilio Credentials

Edit `.env` file and add your Twilio credentials:

```bash
# Get from: https://console.twilio.com
TWILIO_ACCOUNT_SID=ACxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
TWILIO_AUTH_TOKEN=your_auth_token_here

# Twilio WhatsApp Sandbox (testing)
TWILIO_WHATSAPP_NUMBER=whatsapp:+14155238886

# Your WhatsApp number (for sending)
MY_WHATSAPP_NUMBER=whatsapp:+923001234567

# Enable WhatsApp Watcher
WHATSAPP_ENABLED=true

# Optional: Adjust settings
WHATSAPP_WATCHER_INTERVAL=30
WHATSAPP_MAX_RESULTS=10
```

### 2. Install Dependencies

```bash
pip install twilio
# Or
pip install -r requirements.txt
```

### 3. Test Connection

```bash
# Test single run
python3 whatsapp_watcher.py

# Send test message
python3 whatsapp_watcher.py --send +923001234567 "Test message from Digital Employee"
```

### 4. Start Background Watcher (tmux)

```bash
# Start in background
python3 whatsapp_watcher.py --start

# Check status
python3 whatsapp_watcher.py --status

# Stop watcher
python3 whatsapp_watcher.py --stop
```

## 📁 How It Works

### Workflow

```
┌─────────────────────────────────────────────────────────────────┐
│                    WHATSAPP WATCHER FLOW                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  1. Twilio receives WhatsApp message                            │
│          ↓                                                       │
│  2. whatsapp_watcher.py fetches message (every 30s)             │
│          ↓                                                       │
│  3. Check if already processed (deduplication)                  │
│          ↓                                                       │
│  4. Create task file in /Needs_Action/                          │
│          ↓                                                       │
│  5. Mark message as read in Twilio                              │
│          ↓                                                       │
│  6. Log action to /Logs/whatsapp_watcher_*.log                  │
│          ↓                                                       │
│  7. Orchestrator processes task file                            │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## ⚙️ Configuration Options

### Environment Variables (.env)

| Variable | Description | Default | Required |
|----------|-------------|---------|----------|
| `TWILIO_ACCOUNT_SID` | Twilio Account SID | `""` | ✅ Yes |
| `TWILIO_AUTH_TOKEN` | Twilio Auth Token | `""` | ✅ Yes |
| `TWILIO_WHATSAPP_NUMBER` | Twilio WhatsApp number | `""` | ✅ Yes |
| `MY_WHATSAPP_NUMBER` | Your WhatsApp number | `""` | ❌ Optional |
| `WHATSAPP_ENABLED` | Enable/disable watcher | `false` | ❌ Optional |
| `WHATSAPP_WATCHER_INTERVAL` | Check interval (seconds) | `30` | ❌ Optional |
| `WHATSAPP_MAX_RESULTS` | Max messages per fetch | `10` | ❌ Optional |

### Priority Detection

Messages are auto-categorized based on keywords:

| Priority | Keywords |
|----------|----------|
| **High** | urgent, asap, immediately, important, emergency, critical, deadline, now, right away |
| **Medium** | meeting, schedule, call, review, approval, question, help, update, feedback |
| **Normal** | All other messages |

## 🐛 Troubleshooting

### Issue: "Twilio credentials not configured"

**Solution:**
1. Check `.env` file has correct credentials
2. Verify `TWILIO_ACCOUNT_SID` and `TWILIO_AUTH_TOKEN` are set
3. Test connection: `python3 whatsapp_watcher.py`

### Issue: "No messages found"

**Solution:**
1. Send a message to your Twilio WhatsApp number
2. Check Twilio console for incoming messages
3. Verify `TWILIO_WHATSAPP_NUMBER` is correct

### Issue: "Watcher not starting in tmux"

**Solution:**
1. Install tmux: `sudo apt install tmux`
2. Check if session exists: `tmux list-sessions`
3. Try manual start: `python3 whatsapp_watcher.py --continuous`

### Issue: "Messages not being processed"

**Solution:**
1. Check `WHATSAPP_ENABLED=true` in `.env`
2. Verify orchestrator is running: `python3 orchestrator.py`
3. Check logs for errors

### Issue: "Duplicate messages being created"

**Solution:**
1. Check `Metrics/whatsapp_processed_ids.json` exists
2. Verify file is writable
3. Restart watcher: `python3 whatsapp_watcher.py --stop && python3 whatsapp_watcher.py --start`

## 🎯 Best Practices

1. **Enable Only When Needed:** Set `WHATSAPP_ENABLED=false` when not using
2. **Monitor Logs Daily:** Check for errors in `/Logs/`
3. **Keep Credentials Secure:** Never commit `.env` to Git
4. **Test Before Production:** Use Twilio sandbox first
5. **Set Reasonable Intervals:** 30-60 seconds is optimal
6. **Clean Old Logs:** Logs older than 7 days auto-delete via cron
