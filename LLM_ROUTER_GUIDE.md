## 🚀 Quick Start

### Basic Usage

```python
from llm_router import get_response

# Simple call
result = get_response("Summarize this email thread...")

if result["content"]:
    print(f"✅ Response from {result['model_used']}")
    print(result["content"])
else:
    print(f"❌ All models failed: {result['error']}")
```

### With System Prompt

```python
result = get_response(
    prompt="Draft a professional reply to this email:\n\n{email_body}",
    system_prompt="You are an executive assistant. Write concise, professional emails.",
    temperature=0.7,
    max_tokens=2000,
)

reply_body = result["content"]
print(f"Generated by: {result['model_used']} in {result['latency']}s")
```

### Step 3: Example — Email Reply Generation

```python
def generate_email_reply(email_data: Dict) -> Optional[str]:
    """Generate email reply using LLM router."""
    
    prompt = f"""
    Draft a professional reply to this email:
    
    From: {email_data.get('from', 'Unknown')}
    Subject: {email_data.get('subject', 'No subject')}
    
    {email_data.get('body', '')}
    
    Keep it concise and professional. Sign off as {DEFAULT_SENDER['name']}.
    """
    
    result = get_response(
        prompt=prompt,
        system_prompt="You are an executive assistant for a CTO/AI Engineer.",
        temperature=0.7,
        max_tokens=1500,
    )
    
    if result["content"]:
        logger.info(f"Email reply generated by {result['model_used']} ({result['latency']}s)")
        return result["content"]
    else:
        logger.error(f"Failed to generate email reply: {result['error']}")
        return None
```

def analyze_task(content: str) -> Dict:

## 🔄 How Failover Works

### Example Log Output

```
[2025-04-04 10:15:23] [INFO   ] 🔵 Calling Claude (claude-sonnet-4-20250514)...
[2025-04-04 10:15:25] [ERROR  ] 🔴 Claude failed: rate limit — Error code: 429 - Rate limit exceeded
[2025-04-04 10:15:25] [WARNING] 🟡 ⚡ Claude rate limit → switching to Qwen
[2025-04-04 10:15:25] [INFO   ] 🔵 Calling Qwen (qwen/qwen-plus)...
[2025-04-04 10:15:30] [INFO   ] 🟢 Qwen responded in 5.2s (1247 chars)
```

### What Triggers Failover?

| Error Type | HTTP Code | Triggers Switch? |
|------------|-----------|------------------|
| Rate Limit | 429 | ✅ Yes |
| Quota Exceeded | 402/429 | ✅ Yes |
| Timeout | - | ✅ Yes |
| Server Error | 500/502/503/504 | ✅ Yes |
| Connection Error | - | ✅ Yes |
| Invalid API Key | 401 | ❌ No (won't retry) |
| Invalid Request | 400 | ❌ No (won't retry) |

## 📊 Response Format

Every call returns a dictionary:

```python
{
    "content": "The response text here...",     # str or None
    "model_used": "Claude",                      # str or None
    "model_id": "claude-sonnet-4-20250514",     # str or None
    "attempts": 2,                               # int (how many models tried)
    "latency": 5.23,                             # float (seconds)
    "error": None                                # str or None
}
```

## 🎯 Advanced Usage

### Direct Router Access (For More Control)

```python
from llm_router import LLMRouter, ModelConfig

# Create custom router with specific models
custom_router = LLMRouter(
    models=[
        ModelConfig("Claude", "claude-3-5-sonnet-20241022", "ANTHROPIC_API_KEY"),
        ModelConfig("OpenAI", "gpt-4o-mini", "OPENAI_API_KEY"),
    ]
)

# Use it
result = custom_router.get_response("Hello!")
```

### Reset Failed Models Cache

```python
from llm_router import _get_router

router = _get_router()
router.reset_failures()  # Clear failed models cache (e.g. after cooldown)
```

### Check Which Model to Use for Different Tasks

```python
# High-quality reasoning → Claude (default)
result = get_response("Analyze this complex legal document...", max_tokens=8000)

# Fast response → Any model works
result = get_response("Classify this email as urgent/normal/low", temperature=0.1)

# Creative writing → Adjust temperature
result = get_response("Write an engaging LinkedIn post...", temperature=0.9)
```

## ⚠️ Troubleshooting

### "LiteLLM is not installed"
```bash
pip install litellm
```

### "No API keys configured"
- Check `.env` file exists in vault root
- Verify API keys are correct (no extra spaces)
- Run: `python -c "from dotenv import load_dotenv; load_dotenv(); import os; print(os.getenv('ANTHROPIC_API_KEY', 'NOT SET')[:10])"`

### "All models failed"
- Check internet connection
- Verify API keys have active subscriptions
- Check `Logs/llm_router.log` for detailed error messages
- Try: `python llm_router.py` to run diagnostic

### Slow responses
- First call may be slower (cold start)
- Check if router is falling back to slower models
- Consider increasing `LLM_TIMEOUT` if hitting timeouts

## 🎓 Best Practices

1. **Always check `result["content"]` is not None** before using
2. **Log `result["model_used"]`** to track which models are actually serving
3. **Monitor `result["attempts"]`** — high numbers indicate primary model issues
4. **Use system prompts** for consistent behavior across different models
5. **Keep prompts focused** — router doesn't maintain conversation history
6. **Set appropriate max_tokens** — don't waste tokens on simple tasks
